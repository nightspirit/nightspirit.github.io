Bluetooth communication between Cell phone & Arduino BT

11-25-2009

This video proved the concept of the Bluetooth communication between cell phone and Arduino. In the first generation design, we attempt to use the Cell phone as the central controller therefore how to build up the communication between Arduino and Cell phone become a challenge.

We thought the Arduino BT would be the solution. The onboard BT module saves us  time and energy on hardware implementation. However, the software part tortured me for a week. Neither J2ME nor BT protocol is easy!

After some struggles, I decided to use the simplest BT service which called “SPP - Serial Port Profile”. As well, I didn’t follow the standard procedure to search the UUID but directly write it into the J2ME applet. Yeah! That is a kind of shortcut. On the other hand you lose the elasticity for multiple devices so if your project is using multiple Arduino then there is no way to use this cheat.

On the Arduino side I wrote a famous game “Snake”. The cell phone with my pre-loaded applet became a remote controller. Once you establish the SPP service then the communication is just the simple text message. Map the key event to particular character and send it out was what I do in the applet.

For Arduino, you need to start the serial port at setup phase. Make sure both java applet and arduino sketch has the same transmission rate 115200 Hz. You should keep tracking the incoming serial messages and reacting the specific action for each character command.

The result is as video:

Well, we eventually give up the BT because it is tricky and unstable. We also concern the power consumption and it is not suitable for peer communication. In the second generation design we moved to Xbee and discard the Cell phone. (Trust me, that was totally a bad idea!)
